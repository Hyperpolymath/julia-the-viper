# SITE

## Design Philosophy
    Security by Design - Grammatical impossibility of code injection
    Harvard Architecture - Strict separation of Control and Data
    Totality - Provable termination guarantees
    Offline-First - Works without network
    Type Safety - Memory safety guaranteed

## Project Structure
    Monorepo layout
    Packages: jtv-lang, jtv-analyzer, jtv-safe
    Examples: basic, advanced, contracts, integrations
    Tools: CLI, VS Code extension, LSP (planned)

## Key Features
    7 number systems (Int, Float, Rational, Complex, Hex, Binary, Symbolic)
    Addition-only Data Language
    Turing-complete Control Language
    Smart contract security focus
    Reversible computing ready (v2)

# PHILOSOPHY

    "Code injection isn't prevented by careful programming -
     it's prevented by making it grammatically impossible to express."

    Named after Julia Robinson, mathematician who helped solve
    Hilbert's 10th problem. The "Viper" is a playful pun on "adder"
    (both a snake and a calculator).
